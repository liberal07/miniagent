# 笔试题原始要求

从零实现一个最小可用 Agent，允许使用 AI 工具辅助开发。

## 要求

- 支持多轮对话和 session 维护。
- 不直接依赖现成 Agent 框架完成主流程，例如 LangChain、OpenHands 等。
- 核心 runtime 需要自己实现。
- Agent 至少支持一个基本循环：

```text
接收用户输入
判断是直接回答，还是调用工具
执行工具
读取工具结果
继续下一步，直到给出最终答案
```

- 至少提供 3 个工具，例如：

```text
calculator
search，可以 mock
read_docs / todo / weather，可以自定义
```

- 需要有：

```text
最大步数限制
基本异常处理
工具调用 trace 或执行日志
```

- 至少支持一个“跨轮次继续执行”的场景，例如：

```text
第一轮发起一个任务
Agent 创建任务并记录状态
第二轮用户追问进度
Agent 能基于已有状态继续处理，而不是把每轮都当成全新问题
```

- 需要使用真实 LLM 调用。

## 提交内容

```text
代码链接
终端或网页操作录屏
README，包括运行方式、系统设计、memory 的召回时机与放置方式说明
AI Prompt 与问题解决记录
```
